# Review Checklist

Use this checklist when reviewing proofs — whether reviewing someone else's work or revisiting your own.

## Checklist

- [ ] **Plain-language intent:** Is there a plain-language statement of intent to compare against?
- [ ] **Evil implementations ruled out:** Are the evil implementations from step 4 explicitly ruled out?
- [ ] **Quantifier scopes justified:** Has every quantifier scope been justified?
- [ ] **Boundary/edge cases checked:** Have boundary and edge cases been checked?
- [ ] **Concrete examples first:** Does the theorem follow the concrete examples?
- [ ] **Statements-only read-through:** Has someone read only the statements without reading the proofs?
- [ ] **Trusted base reviewed:** Is the trusted base (axioms, imports, sorries) listed and reviewed separately?
- [ ] **Sorries tracked:** Are there any `sorry`s, and is each one tracked as a known gap?
